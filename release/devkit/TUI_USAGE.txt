==========================================================
  EFL TUI — Terminal User Interface
==========================================================

The EFL TUI is a display-only diagnostic monitor. It connects to the
running EFL engine via a named pipe and shows live framework state.
There is no interactive input — only Ctrl+C to stop.

LAUNCHING
  Run efl-tui.exe from any terminal (cmd, PowerShell, Windows Terminal).
  The TUI will wait for the game to launch and connect automatically.

  Alternatively, EFL can spawn the TUI as a child process if efl-tui.exe
  is located in the game folder or on your PATH.

THREE PHASES
  1. BOOT — Animated startup sequence showing version checks, manifest
     loading, and subsystem initialization as they happen.

  2. DIAGNOSTICS — Structured validation report with a card grid of
     coded errors. Each code follows the format CATEGORY-SEVERITY###:
       Categories: BOOT, MANIFEST, HOOK, AREA, WARP, RESOURCE, NPC,
                   QUEST, TRIGGER, STORY, SAVE, IPC
       Severities: E (Error), W (Warning), H (Hazard)
       Example:    MANIFEST-E001 — missing required field "modId"

  3. MONITOR — Live dashboard showing active hooks, recent events,
     save operations, and per-mod status. Updated in real time via
     the named pipe.

IPC PROTOCOL
  The engine writes JSON Lines to: \\.\pipe\efl-{pid}
  The TUI reads passively. If the TUI is not running, the engine
  continues normally — the pipe is best-effort, never blocking.

VISUAL STYLE
  Dark background with neon accent colors (green, amber, magenta, cyan).
  Bordered panels, colored stat bars, all-caps headers, pill/badge labels.
  Best viewed in Windows Terminal with a dark theme.

TROUBLESHOOTING
  - If the TUI shows "waiting for pipe..." indefinitely, make sure:
    - The game is running
    - EFL.dll is loaded (check <game folder>\EFL\logs\)
  - The TUI requires a terminal with ANSI color support.
    Windows Terminal and modern PowerShell work. Legacy cmd.exe may not.
